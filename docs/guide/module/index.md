# 模块专题

这里保留的是历史模块专题入口。当前建议优先从 [低代码开发](../../low-code/)、[运维管理](../../operations/) 或 [核心概念](../../concepts/) 主线进入，再按需要回到本目录查阅具体模块说明。

本目录更适合在你已经明确要查看某个模块时使用，而不是作为第一次认识平台的阅读起点。它更像“模块能力索引”，而不是完整学习路径。

## 建议优先阅读

1. [低代码开发](../../low-code/)
2. [运维管理](../../operations/)
3. [核心概念](../../concepts/)

## 这目录适合什么时候用

当你已经知道自己要处理的是某类平台模块，而不是一个完整业务交付任务时，这里会更好用。

典型场景包括：

- 我现在就要做审批流
- 我现在要补组织架构和员工关系
- 我现在要补用户、角色和访问边界
- 我现在要做通知通道或通知模板

如果你的问题还是“我应该按什么顺序交付一个业务功能”，优先回到 [低代码开发](../../low-code/)。

## 一条更实用的使用方式

大多数情况下，建议这样使用本目录：

1. 先在主线里确认当前任务属于哪一类交付阶段
2. 再回到这里打开具体模块页
3. 用模块页解决操作细节、页面入口和边界问题
4. 做完之后，再回到主线继续推进下一个阶段

这样可以避免一开始就在模块专题之间来回跳，却不知道整体交付顺序。
