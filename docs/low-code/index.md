# 低代码开发

适用对象：主要使用平台内建能力完成业务交付的业务开发人员。

这部分内容围绕“如何用平台完成业务需求”组织，而不是按底层模块名平铺。你可以从应用创建开始，逐步进入业务模型、页面表单、流程、身份/访问边界和通知。

如果你第一次按平台主线交付业务功能，建议先把下面两篇读完，再进入具体专题页：

## 平台补充能力

除了主线里的模型、流程和权限能力，平台还内建了一些常被忽略但很实用的业务支撑能力，例如动态表单、文件上传和文档管理。你可以先看 [平台能力地图](../reference/platform-capability-map)，再判断哪些能力可以直接纳入当前方案。

如果你正好在处理采集表单、业务附件或共享资料库，建议继续看 [表单、附件与文档能力](./forms-attachments-and-documents)。

如果你已经做到了模型、页面或流程，但准备补齐上线所需的组织、身份、访问边界、参数和消息能力，建议继续看 [组织、权限、配置与通知](./organization-permissions-configuration-and-notification)。

如果你正在判断审批流、逻辑流、表达式、脚本和通知该怎么分工，建议继续看 [流程、规则与自动化](./workflow-rules-and-automation)。

如果你正在判断标准页面、动态表单和自定义 UI Schema 该怎么选，建议继续看 [页面交付：标准页面、动态表单与 UI Schema](./page-delivery-with-dynamic-forms-and-ui-schema)。

如果你在做首页、工作台或数据看板，也可以先补看 [工作台、Widget 与平台组件库](../fusion-development/workbench-widgets-and-component-libraries)，很多场景还不需要直接做独立前端。
