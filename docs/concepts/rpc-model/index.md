# RPC 模型

适用对象：需要对接第三方系统、外部服务和企业内部平台的开发人员。

RPC 模型用于把“如何调用外部系统”沉淀成统一配置。它把协议、认证、参数、返回结构和调试方式集中管理，降低接口集成时的重复劳动。

## 它解决什么问题

- 把不同来源的远程接口统一纳入平台模型管理
- 降低重复编写接入代码和调试脚本的成本
- 让外部集成可以被逻辑流、业务模型和自定义代码稳定复用

## 核心组成

- 目标服务：接口名称、地址、协议
- 认证方式：鉴权、签名、凭据传递
- 参数与返回：请求参数、返回结构、异常信息
- 调试与日志：调用测试、问题定位和调用记录

## 与其他模型的关系

- [逻辑流模型](../logicflow-model/) 经常会调用 RPC 模型对接外部系统
- [业务模型](../business-model/) 对外协同的业务动作可能依赖 RPC 能力
- 融合开发可以进一步扩展协议适配、鉴权和通用集成能力

## 典型使用场景

- 对接 ERP、OA、短信、支付、地图、设备平台等第三方系统
- 把常用远程调用沉淀成组织内部可复用能力
- 在低代码业务流中稳定调用外部接口
