# UI 模型

适用对象：负责页面入口设计的低代码开发人员、前端扩展开发人员。

UI 模型定义“用户会看到哪个页面入口，以及这个入口由什么方式实现”。它主要管理路由地址、页面标题和渲染方式，是页面层的总开关。

## 它解决什么问题

- 把页面入口从具体实现里抽出来统一管理
- 允许同一个业务入口在不同阶段切换实现方式
- 为标准页面、UI Schema 页面和微前端页面提供统一编排点

## 核心组成

- 路由地址：页面在前端的访问路径
- 标题与元信息：导航显示名、标签等
- 渲染方式：内部页面、[UI Schema](../ui-schema/) 或微前端
- 页面参数：页面打开时需要携带的上下文或配置

## 与其他模型的关系

- [菜单模型](../menu-model/) 往往通过菜单项把 UI 模型暴露给用户
- [权限模型](../authority-model/) 会决定用户是否有权访问某个 UI 模型
- UI 模型可以由 [业务模型](../business-model/) 自动生成默认入口，也可以在融合开发中被覆写

## 典型使用场景

- 为业务对象生成列表页、详情页、表单页入口
- 把标准页面替换成专用的 UI Schema 页面
- 把标准入口升级为微前端实现，以承接复杂交互
