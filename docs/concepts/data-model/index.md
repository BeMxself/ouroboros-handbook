# 数据模型

适用对象：低代码开发人员、后端扩展开发人员、需要统一管理数据结构和查询语义的人员。

数据模型是潮汐栈对业务数据结构、字段关系、校验规则和持久化行为的统一描述。它承接“数据长什么样、如何存、如何查、如何校验”这些基础问题。

## 它解决什么问题

- 统一定义业务数据结构，避免数据库设计和业务配置脱节
- 用一致的校验、主键和查询能力支撑模型运行时
- 为低代码生成、逻辑流处理和接口访问提供稳定的数据基础

## 核心组成

- 模型基本信息：模型名、标题、存储位置、说明等
- 字段定义：字段类型、是否必填、默认值、校验规则
- 关系定义：对象之间的关联方式
- 主键与编码：主键生成策略、业务编码等
- 查询语义：筛选、排序、分页和查询 DSL 的基础能力

## 与其他模型的关系

- 数据模型经常由 [业务模型](../business-model/) 投射出来，也可以独立定义
- [字段定义](../field-define/) 用来沉淀可复用字段类型
- [逻辑流模型](../logicflow-model/) 和 [WebAPI 模型](../webapi-model/) 往往围绕数据模型做增删改查
- 后端融合开发可以在此基础上增加自定义字段类型、查询或数据处理逻辑

## 典型使用场景

- 定义客户、订单、库存、设备、工单等业务对象的数据结构
- 支撑标准 CRUD、查询分析和列表展示
- 作为逻辑流、审批流程和接口入参与出参的核心数据载体
