# 权限模型

适用对象：应用管理员、业务开发人员、负责访问控制和安全设计的专业开发者。

权限模型定义“谁能访问什么，以及在什么条件下访问”。它把页面、菜单、接口和数据范围相关的控制规则统一成平台可理解的权限对象。

## 它解决什么问题

- 统一组织角色授权，而不是分别在页面和接口里重复做控制
- 把访问控制和业务能力装配到同一套模型体系里
- 为复杂的数据权限和菜单权限预留清晰的治理入口

## 核心组成

- 权限标识：权限名、编码和说明
- 绑定目标：页面、接口、菜单、业务动作
- 控制参数：角色、组织、数据范围、上下文条件
- 访问结果：可见、可用、可操作的最终控制效果

## 与其他模型的关系

- [菜单模型](../menu-model/) 和 [UI 模型](../ui-model/) 负责暴露入口，权限模型决定入口能否被使用
- [WebAPI 模型](../webapi-model/) 负责接口契约，权限模型决定接口能否被调用
- [模块模型](../module-model/) 负责组织功能结构，权限模型负责治理使用边界

## 典型使用场景

- 根据角色控制页面、菜单和接口访问
- 根据组织、岗位或数据范围做细粒度授权
- 在同一应用里区分管理员、业务人员、审核人员、访客等不同视角
