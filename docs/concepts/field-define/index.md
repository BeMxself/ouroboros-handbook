# 字段定义

适用对象：维护平台字段规范的人、低代码建模人员、需要扩展字段能力的专业开发者。

字段定义用于沉淀“一个字段类型应该怎么存、怎么校验、怎么展示”的公共约定。它是业务模型和数据模型复用字段能力的基础，而不是某个具体业务表上的单个字段实例。

## 它解决什么问题

- 避免同一类字段在不同模型里重复配置
- 统一字段的存储类型、校验规则和默认展示方式
- 为行业化场景扩展专用字段类型提供稳定入口

## 核心组成

- 字段标识与标题：字段类型的唯一名称和展示名称
- 数据语义：底层值类型、可选范围、默认值
- 校验约束：长度、格式、数值范围、必填等
- 展示约定：表单控件、列表呈现、交互行为
- 扩展实现：必要时通过代码补充专用逻辑

## 与其他模型的关系

- [数据模型](../data-model/) 和 [业务模型](../business-model/) 会直接引用字段定义
- 自定义字段能力通常通过后端扩展来提供
- 页面渲染层会结合 [UI Schema](../ui-schema/) 或默认渲染规则来消费字段定义

## 典型使用场景

- 统一“金额”“手机号”“行政区划”“身份证号”等通用字段
- 为行业场景增加专用字段，如“设备编号”“项目编码”“合同金额”
- 让低代码建模时直接复用组织内沉淀好的字段规范
