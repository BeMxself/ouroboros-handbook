# 菜单模型

适用对象：负责应用导航设计的人，以及需要组织页面入口和功能层级的开发者。

菜单模型是潮汐栈里对导航结构的统一定义。它决定用户在应用里如何看到各项功能入口，以及这些入口以什么层级、名称和图标展示。

## 它解决什么问题

- 让导航结构不必硬编码在前端工程里
- 让页面入口可以随着业务模型和模块结构统一调整
- 为多角色、多应用和多租户场景提供可治理的菜单编排方式

## 核心组成

- 菜单名称与标题：系统内标识和用户可见名称
- 图标与排序：导航展示效果
- 路由与跳转目标：通常关联 [UI 模型](../ui-model/)
- 树形结构：父子层级、分组和嵌套关系
- 可见性控制：与 [权限模型](../authority-model/) 协同

## 与其他模型的关系

- 菜单模型主要把 [UI 模型](../ui-model/) 暴露为可访问导航
- [模块模型](../module-model/) 用来组织更高层级的功能结构
- [权限模型](../authority-model/) 控制哪些菜单项对哪些角色可见

## 典型使用场景

- 按业务域组织应用菜单
- 为不同角色显示不同导航
- 在不改前端代码的前提下调整系统菜单结构
