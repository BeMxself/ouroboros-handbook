# 逻辑流模型

适用对象：需要编排业务处理逻辑的低代码开发人员，以及希望把复杂业务逻辑纳入平台运行时的专业开发者。

逻辑流模型是潮汐栈里承接“机器执行逻辑”的模型。它基于节点、端口和连线来组织执行过程，适合把数据处理、校验、条件分支、调用服务和持久化操作组合成一条可视化流程。

## 它解决什么问题

- 让复杂业务逻辑不必全部写成手工代码
- 用图形化方式组织节点处理，降低理解和协作成本
- 让平台模型、脚本、表达式、数据访问能力以统一方式协同

## 核心组成

- 模型基本信息：模型名、说明、输入输出约定
- 节点定义：每个节点承担单一职责，如校验、映射、查询、脚本执行
- 连接定义：数据在节点之间如何流转
- 运行时属性：异步、错误处理、调试和执行上下文

## 与其他模型的关系

- [WebAPI 模型](../webapi-model/) 可以把接口处理适配到逻辑流
- [业务模型](../business-model/) 的业务操作常常会落到逻辑流实现
- [数据模型](../data-model/) 提供逻辑流中的数据读写基础
- [表达式上下文](../../api/backend/expression-context/) 和 [脚本上下文](../../api/backend/script-context/) 为节点计算提供能力

## 典型使用场景

- 表单提交后的数据校验、补充和保存
- 按条件分支执行不同业务规则
- 在一个请求内串联数据查询、第三方调用和结果返回

## 它和审批流模型的区别

- 逻辑流模型更偏“短事务、机器执行、即时返回”
- [审批流模型](../approval-workflow-model/) 更偏“长流程、多人参与、跨时段推进”
