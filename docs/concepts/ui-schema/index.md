# UI Schema

适用对象：主要做低代码页面搭建的人，以及需要理解潮汐栈页面 DSL 的专业开发者。

UI Schema 是潮汐栈里描述页面结构、组件、数据绑定和交互行为的声明式 DSL。平台当前基于 [amis](https://github.com/baidu/amis) 渲染这类页面，因此它特别适合标准化、结构化的业务界面。

## 它解决什么问题

- 让表单、列表、详情、看板等页面可以通过配置而不是完整前端工程来交付
- 让页面结构和业务数据、接口调用之间建立稳定的声明式关系
- 为低代码和融合开发之间提供一个清晰的协作边界

## 核心组成

- Schema 结构：页面、布局、组件树
- 数据绑定：组件与接口、上下文、变量之间的关系
- 行为配置：按钮动作、表单提交、联动、条件显示
- 组件扩展：在标准组件基础上扩展自定义能力

## 与其他模型的关系

- [UI 模型](../ui-model/) 决定某个页面入口是否由 UI Schema 实现
- [业务模型](../business-model/) 往往会生成基础页面结构
- [WebAPI 模型](../webapi-model/) 为页面提供数据读取和提交能力
- 当标准组件不足时，可以转向融合开发扩展前端能力

## 典型使用场景

- 标准 CRUD 页面
- 审批单、业务表单、列表查询和详情展示
- 以配置为主的门户、看板和辅助页面
