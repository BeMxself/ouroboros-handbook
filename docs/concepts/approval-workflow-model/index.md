# 审批流模型

适用对象：需要在业务功能中引入申请、审批、驳回、流转等长流程机制的人员。

审批流模型用于定义“人参与的业务流程”。它强调的是流程节点、参与人、流转规则和过程状态，而不是一次请求内完成的即时逻辑。

## 它解决什么问题

- 把请假、报销、采购申请、合同审批等长流程业务标准化
- 让流程定义、发起、处理和统计围绕同一套模型运行
- 把业务对象和审批过程有机串联，而不是依赖分散的临时实现

## 核心组成

- 流程定义：流程名称、适用范围、说明
- 审批节点：审批人、审批规则、分支和回退
- 流程状态：发起、处理中、结束、驳回、撤回等
- 统计与追踪：谁处理过、当前卡在哪里、流程整体表现如何

## 与其他模型的关系

- [业务模型](../business-model/) 往往作为审批对象或业务载体
- [权限模型](../authority-model/) 决定谁能发起、处理和查看审批
- 消息通知、页面入口和统计能力通常围绕审批流模型展开
- 它与 [逻辑流模型](../logicflow-model/) 的区别在于，审批流处理的是“人参与的长流程”

## 典型使用场景

- 请假、报销、用印、采购、立项、合同等审批业务
- 需要按组织关系、岗位或条件分支进行审批的场景
- 需要统计审批效率、节点耗时和流程状态的场景
